# 前端架构规则

## 总体数据流

功能代码默认遵循单向依赖：

~~~text
页面或组件
    -> 组合式函数或 Pinia Store
    -> API 模块
    -> Alova 传输层
    -> FastAPI
~~~

禁止页面直接拼接多个接口、直接管理全局令牌、直接解析复杂后端响应或直接动态导入服务端组件。

## 目录职责

| 目录 | 职责 | 禁止承担的职责 |
| --- | --- | --- |
| src/api | 接口地址、请求 DTO、响应 DTO、传输适配 | 页面展示、路由跳转、权限猜测 |
| src/components | 可复用展示组件和交互组件 | 复杂业务请求和全局会话 |
| src/composables | 可复用页面行为、加载和生命周期 | 直接定义跨模块权限策略 |
| src/constants | 稳定常量和枚举映射 | 保存令牌和运行时用户数据 |
| src/layouts | 布局、菜单容器、页面壳 | 具体资源的 CRUD 业务逻辑 |
| src/router | 静态路由、守卫、动态路由注册 | 代替后端授权 |
| src/stores | 会话、用户、权限和跨页面状态 | 直接访问 DOM 或保存服务端密钥 |
| src/types | 共享类型和运行时类型守卫 | 发起网络请求 |
| src/utils | 纯函数、格式化和边界适配 | 隐式导航、隐式请求和全局副作用 |
| src/views | 路由级页面和页面编排 | 重复实现传输、权限和通用状态 |

允许页面调用组合式函数或 Store，允许组合式函数调用 API 模块，禁止反向依赖。

## 传输层

传输层必须集中处理以下能力：

1. API 基地址和版本前缀。
2. Authorization 请求头。
3. JSON 响应统一解包。
4. HTTP 状态和业务 code 的归一化。
5. 超时、取消和有限重试。
6. 401 的单飞刷新和失败退出。
7. JSON、文本、文件和流式响应的模式区分。

业务 API 模块只描述接口和 DTO，不在每个页面重复写鉴权、解包和错误转换。

JSON 合同：

~~~ts
type ApiResponse<T> = {
  code: number
  error_code?: string | null
  message: string
  data: T | null
}
~~~

遇到 204、文件、流、HTML 或明确跳过包装的接口，必须显式选择对应响应模式。

## 会话启动顺序

登录后或刷新页面初始化会话时，使用以下顺序：

1. 读取内存中的会话状态；没有访问令牌时进入登录流程。
2. 调用当前用户信息接口，确认用户状态和权限。
3. 处理 must_change_password，必要时只允许进入修改密码页面。
4. 调用动态路由接口。
5. 校验路由结构、组件名、重定向和子路由。
6. 通过本地组件白名单把动态路由转换为 Vue Router 路由。
7. 注册路由后再进入业务页面。

会话初始化必须可重复调用且有去重机制，不能并发创建多个初始化流程。

## 状态管理

会话 Store 至少区分：

- 未初始化。
- 初始化中。
- 已登录。
- 需要修改密码。
- 已退出。
- 初始化失败。

Store 负责访问令牌、刷新令牌的生命周期、当前用户、权限集合和会话清理。页面只通过 Store 暴露的动作和计算状态使用会话，不直接操作浏览器存储或请求头。

## 动态路由

- 服务端返回的数据必须先通过运行时类型守卫。
- component 只能匹配本地 import.meta.glob 或显式注册表。
- path、name 和 redirect 必须限制在应用允许的路由格式内。
- 重复路由名、未知组件、危险重定向和坏的 children 必须被拒绝或隔离。
- 外链和 iframe 必须经过来源白名单校验。
- 动态路由加载失败时保留可用的登录、错误和无权限页面，不让整个应用进入不可恢复状态。

## 页面状态

数据页面至少明确以下状态：

- 初始状态。
- 加载中。
- 有数据。
- 空数据。
- 参数校验失败。
- 401 或会话失效。
- 403 或无权限。
- 限流。
- 网络错误。
- 可重试失败。

按钮禁用、加载提示和页面隐藏只改善体验，不得用来代替服务端安全检查。

## 文件和导出

- 上传组件只做用户体验层的大小和扩展名提示。
- 后端负责文件内容签名、病毒扫描、租户、所有者和权限检查。
- 下载和导出接口使用 blob、arrayBuffer 或流式响应模式，并从响应头或后端元数据读取文件名。
- 对象 URL 用完立即释放。
- 不能把受保护文件 URL 直接写入公开页面或长期浏览器存储。
