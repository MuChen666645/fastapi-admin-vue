# 前端 Codex 核心文件

本目录保存 Vue 前端项目的专属规则，用于保证 Codex 或其他代码代理修改仓库时，始终遵守统一的范围、架构、类型和安全要求。

## 阅读顺序

开始实现前，必须按以下顺序阅读：

1. `AGENTS.md`：强制执行的工程规范和修改边界。
2. `project-context.md`：从当前前端和 FastAPI 服务确认的项目事实。
3. `security-boundary.md`：鉴权、权限、浏览器数据、文件和不可信配置的安全规则。
4. `checklist.md`：开发验证和交付前检查清单。
5. `PROMPTS/bugfix.md` 或 `PROMPTS/feature.md`：缺陷修复或功能开发任务模板。

`../AGENTS.md` 是 `frontend/` 目录的自动加载入口，并指向本目录的规则文件。

## 规则优先级

- 用户明确提出的需求和范围优先级最高。
- 当本文档与实际代码或后端 DTO、路由不一致时，以经过核对的代码和契约为准，并在任务结束时更新本文档。
- 本目录只定义前端行为和协作边界，不授权修改 `../service/`。
- 规则文件不能保存密钥、令牌、生产数据、生成物、临时日志或调试输出。

## 明确不做的事情

- 不在前端重新实现后端鉴权、授权、租户隔离、参数最终校验、文件安全检查或限流。
- 不为了完成页面而静默修改后端契约。
- 不借助本目录的规则引入无必要依赖或进行无关重构。
