# 前端交付检查清单

## 修改前

- 阅读 `.codex/AGENTS.md`、`.codex/project-context.md` 和 `.codex/security-boundary.md`。
- 修改契约相关功能前，检查目标组件、同级组件、路由或 Store 使用方式，以及后端 DTO 和控制器。
- 确认当前任务是纯前端任务，还是明确授权了前后端联动修改。
- 明确加载、空数据、校验失败、未授权、服务端错误、重试和取消状态。

## 修改中

- API 请求只能位于共享传输层或 API 模块，不能直接写在模板中。
- API 边界保持后端蛇形字段命名不变。
- 对不可信响应使用运行时校验，对动态组件和外部 URL 使用白名单。
- 令牌、密码、MFA 值、图形验证码、重置令牌、预签名 URL 和生产数据不得进入日志或浏览器存储。
- 涉及 API 适配、会话转换、权限判断、路由校验或二进制响应时，添加针对性单元测试。
- 生产路径不得创建 Mock 成功回退。

## 交付前

在 `frontend/` 目录执行：

```sh
pnpm type-check
pnpm test:unit -- --run
pnpm build
```

另外执行：

```sh
git diff --check
git status --short
```

确认以下事项：

- 只修改了预期的前端文件；
- 没有新增生成文件、密钥、令牌或生产数据；
- API 响应和字段名与已核对的后端契约一致；
- 没有把前端守卫伪装成安全授权；
- 动态路由和外部 URL 已校验并使用白名单；
- 对象 URL、定时器、监听器和未完成请求已清理；
- 明确报告验证失败和环境限制。
