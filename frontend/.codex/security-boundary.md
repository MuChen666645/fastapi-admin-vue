# 前端安全边界

浏览器是一个不可信执行环境。前端代码只能改善用户体验、减少误操作，不能替代后端安全控制。

## 信任模型

- 用户输入、查询字符串、路由参数、上传文件、响应 JSON、菜单元数据、预签名 URL 和解码后的令牌声明，在校验前都必须视为不可信。
- 身份、权限、租户范围、字段可见性、最终校验、限流、文件检查和审计行为以后端为准。
- 隐藏按钮、禁用控件、前端路由守卫、角色名称或本地权限数组都不能视为授权。

## 凭据和敏感数据

- 禁止提交或展示密钥、生产数据、密码、MFA 验证码、图形验证码、重置令牌、访问令牌、刷新令牌和内部 URL。
- 禁止把密钥放进 `VITE_*`，因为这些值会被打包并暴露给浏览器。
- 访问令牌默认只保存在内存中。避免使用 `localStorage`；只有产品明确要求刷新页面后保留会话时，才允许使用范围最小的 `sessionStorage`，并记录风险。
- 退出登录、刷新失败、切换账号和无法恢复的 401 响应后，必须清理凭据和会话状态。
- 日志、URL、分析数据、错误上报、截图、测试夹具和测试名称中不得包含敏感值。
- 不得根据 JWT 解码结果做权限决策，必须使用服务端确认的用户信息和权限。

## XSS 和导航

- 禁止使用 `v-html`、`innerHTML`、动态脚本注入和任意组件解析。
- 用户控制的内容默认按文本渲染。确实需要富文本时，必须使用获批准的清理器，并为信任边界添加专项测试。
- 重定向地址、外链和 iframe 来源必须匹配明确白名单，禁止把未经校验的值传给 `window.open` 或导航 API。
- 新窗口链接使用 `noopener,noreferrer`。
- 禁止把敏感值放进查询字符串或路由参数。

## API 和会话行为

- 统一处理鉴权请求头、响应解包、错误归一化、超时和令牌刷新。
- 同一时间最多执行一次刷新。刷新失败后必须结束本地会话，禁止无限重试。
- 下载和导出保留原始二进制响应处理，不能仅根据请求 URL 决定是否解析 JSON。
- 短期预签名 URL 必须按敏感凭据处理，不得持久化或记录。
- 不得用伪造成功状态隐藏 401、403、校验失败、限流或服务端错误。

## 文件和浏览器存储

- 客户端文件类型和大小检查只是用户体验提示，不是安全控制。
- 不得信任文件名、扩展名、MIME 类型或隐藏表单值，最终检查必须由后端完成。
- 每次使用或替换对象 URL 后都必须释放它。
- 除非经过明确评审，不得持久化敏感表单草稿、个人数据、重置令牌或 MFA 恢复码。
