# 前端 Codex 入口

修改 frontend/ 下的任何文件前，先确认已经读取仓库根目录 AGENTS.md，再按以下顺序阅读前端规则：

1. .codex/AGENTS.md：强制执行的实现、架构和边界规则。
2. .codex/PROJECT.md：当前前端结构和后端接口契约事实。
3. .codex/ARCHITECTURE.md：前端分层、状态、传输和路由架构。
4. .codex/BOUNDARY.md：前端修改、安全和跨项目边界。
5. .codex/WORKFLOW.md：任务分析、实现、测试和交付流程。
6. .codex/PROMPTS/bugfix.md 或 .codex/PROMPTS/feature.md：缺陷修复或功能开发模板。
